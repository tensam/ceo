# Indie Developer Commercialization Playbook

## The Minimal Path from 0 to 1

### Phase 0: Validation (1-2 Weeks)
- [ ] Describe your product in one sentence (elevator pitch)
- [ ] Talk to 5 potential users (not friends or family)
- [ ] Confirm someone is willing to pay for this
- [ ] Choose a product name and domain

### Phase 1: Landing Page (1 Week)
- [ ] Deploy a landing page (Vercel/Netlify + simple HTML or Next.js)
- [ ] Include: one-line value proposition + screenshot/demo + email signup
- [ ] Set up basic analytics (Plausible/Umami, privacy-friendly)
- [ ] Post to Twitter/X and relevant communities

### Phase 2: Early Users (2-4 Weeks)
- [ ] Invite 10-20 early users for free trial
- [ ] Collect feedback (what works / what doesn't / what's missing)
- [ ] Iterate on the product quickly
- [ ] Start writing 1-2 content pieces (blog post / Twitter thread)

### Phase 3: Start Charging (Month 2)
- [ ] Set pricing (keep it simple, 2 tiers is enough)
- [ ] Integrate payments (Stripe / LemonSqueezy / Paddle)
- [ ] Launch self-serve signup
- [ ] Notify early users

### Phase 4: Growth (Month 3+)
- [ ] Consistent content marketing (1 piece per week)
- [ ] SEO optimization
- [ ] Consider a Product Hunt launch
- [ ] Build a referral mechanism

## Pricing Quick Reference

### Recommended Starting Prices

| Product Type | Suggested Starting Price | Notes |
|-------------|------------------------|-------|
| Developer Tool SaaS | $9-29/mo | Developers are price-sensitive but pay for efficiency |
| B2B SaaS | $29-99/mo | Businesses have larger budgets |
| Productivity Tool | $5-15/mo | Individual users are price-sensitive |
| API Service | Usage-based | Free tier + pay for overages |
| One-time Tool | $19-79 | Lifetime deals can attract early users |

### Pricing Psychology

- **Don't go free** — Free users don't give feedback; paying users are real users
- **Don't go too cheap** — A $5/mo product signals low value
- **Offer annual discounts** — Typically 20%, helps cash flow
- **Start high** — Lowering prices is easy; raising them is hard

## Channel Priority (Indie Developers)

### Tier 1: Must-Do (Free + High ROI)
1. **SEO** — Write articles that solve user problems
2. **Twitter/X** — Build personal brand + Build in Public
3. **Product Hunt** — One-time high exposure

### Tier 2: Recommended (Moderate Investment)
4. **Reddit / HN** — Provide value in relevant communities
5. **Email list** — Attract subscribers with content
6. **Free tools** — Trade coding ability for traffic

### Tier 3: Situational (Requires More Investment)
7. **YouTube** — Tutorials / demo videos
8. **Podcast interviews** — Borrow someone else's audience
9. **Paid ads** — Wait until PMF is confirmed

## Common Mistakes

| Mistake | Better Approach |
|---------|----------------|
| Spend 3 months finding the perfect name | Spend 1 day picking a good-enough name and start |
| Build 5 features before launching | Build 1 core feature and launch |
| Wait until "ready" to promote | Promote while building |
| Build a product for everyone | Target one specific audience |
| Don't charge to validate demand | Start charging as early as possible |
| Only write code, no marketing | 50% product, 50% promotion |
