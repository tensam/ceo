# Naming Reference Examples

## Successful SaaS Product Naming Analysis

### Descriptive Naming
| Product | Meaning | Why It Works |
|---------|---------|-------------|
| Salesforce | Sales + Force | Directly describes purpose |
| YouTube | Your + Tube (channel) | Implies user ownership |
| Mailchimp | Mail + Chimp | Memorable + fun |
| Grammarly | Grammar + ly | Function is clear |

### Suggestive Naming
| Product | Implied Meaning | Why It Works |
|---------|-----------------|-------------|
| Slack | Relaxation / ease | Suggests effortless communication |
| Notion | Idea / concept | Suggests a thinking tool |
| Linear | Direct / streamlined | Suggests efficient workflows |
| Vercel | Accelerate | Suggests speed |

### Coined Naming
| Product | Word Construction | Why It Works |
|---------|-------------------|-------------|
| Spotify | Spot + ify | Unique and registrable |
| Figma | Figure + ma | Short and memorable |
| Calendly | Calendar + ly | Function is inferrable |
| Zapier | Zap + ier | Suggests fast automation |

## Indie Developer Product Naming Trends

### Common Patterns
1. **Verb + ly/fy/io**: Simplify, Convertify
2. **Adjective + Noun**: SuperList, LinearApp
3. **Two-syllable coined words**: Raycast, Posthog
4. **Abbreviations**: Cal.com (Calendar)

### Domain Strategy
- `.com` is still the best (highest trust)
- `.io` suits developer tools
- `.app` suits mobile products
- `.dev` suits developer tools
- `get[name].com` is a good alternative
- `[name]hq.com` is also common

## Tagline Patterns

### Effective Formulas
1. **[Verb] + [Object] + [Method]**: "Build websites visually"
2. **[Noun] for [Users]**: "Analytics for developers"
3. **The [Adjective] way to [Action]**: "The fastest way to build APIs"
4. **[Verb] without [Pain Point]**: "Deploy without the DevOps"
5. **[Number] + [Value]**: "One tool for all your marketing"
